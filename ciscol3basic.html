Quick Reference: Understanding VLANs

A VLAN is a switched network that is logically segmented by function, project team, or application, without regard to the physical locations of the users. VLANs have the same attributes as physical LANs, but you can group end stations even if they are not physically located on the same LAN segment. Any switch port can belong to a VLAN, and unicast, broadcast, and multicast packets are forwarded and flooded only to end stations in the VLAN. Each VLAN is considered a logical network, and packets destined for stations that do not belong to the VLAN must be forwarded through a router or a switch supporting fallback bridging, as shown in Figure 13-1. Because a VLAN is considered a separate logical network, it contains its own bridge Management Information Base (MIB) information and can support its own implementation of spanning tree.  Configuration steps for VLAN2 (SVI), trunked on fa0/13, One Subnet -> 10.0.1.0/24  

						
  Network Diagram of this configuration.  
						 
 VLAN logically creates broadcast domains on the L2 network, so a switch can send traffic only within the same broadcast domain, in otherwords one VLAN. To communicate to devices in other broad cast domain essentially a router is required. Switch Virtual Interface is nothing but acts on Layer-3 to get this routing done, so that an 
						 inter VLAN routing is possible. So VLAN on L2 layer keeps traffic within its own broad cast domain, but to send the traffic to other VLAN's a mechanism is required to route the traffic between these two VLAN's on the L3 layer. This is where SVI comes in, with SVIs the switch recognizes the packet destinations that are local to the sending VLAN and switches those packets and packets destined for different VLANs are routed. A VLAN is mapped usually to one SVI, also unless the VLAN is created there will be no spanning tree instance running hence the line protocol will be down for the SVI VLAN. Source : Cisco Learning Network.
