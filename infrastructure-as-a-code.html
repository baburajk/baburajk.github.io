Network Planning 

 

Derive a simple/small network planning.

1. Split the VCN's into 3 : Public, Private and Management. Public for web-tier/routing, private for the app/db/midtier and management for devops(tooling/ansible)

2. Use smaller subnets from 192.168.0.0/16 from RFC1918 space for PUBLIC, 10.0.0.0/8 for PRIVATE and 172.16.0.0/12 for MANAGEMENT.

3. Use/map the subnets as 192.168.1.0/24 for REGION-1, 192.168.2.0/24 for REGION-2. Likewise for PRIVATE and MANAGEMENT subnets.

baburaj@Aurora-5:~/aurora/work/oci/terraform/devlab/networking/us-ashburn-1\> for region in `seq 1 3`; do echo "PUBLIC VCN for REGION $region"; ipcalc 192.168.${region}.0/24 | grep "Network:" | awk -F":" '{print $2}' | awk '{print $1}' ; echo ; done
